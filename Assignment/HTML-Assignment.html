Q.9 : How is Cell Padding different from Cell Spacing ?

Ans. : In HTML, cell padding and cell spacing are both attributes used in table elements to control the layout and appearance of the table cells, but they have different functions.

Cell padding refers to the space between the content of a table cell and its border. It determines how much space is added to the top, bottom, left, and right of the cell content. You can set the cell padding using the CSS "padding" property or the HTML "cellpadding" attribute.

Cell spacing, on the other hand, refers to the space between adjacent cells in a table. It determines the amount of space between the borders of two neighboring cells, both horizontally and vertically. You can set the cell spacing using the CSS "border-spacing" property or the HTML "cellspacing" attribute.


To summarize, cell padding is the space between the content and the border of a cell, while cell spacing is the space between adjacent cells in a table.


Q.10 : How can we club two or more rows or columns into a single row or column in an HTML table?

Ans. : To merge two or more rows into a single row: Identify the cell that you want to span across multiple rows. Add the rowspan attribute to that cell, and set its value to the number of rows you want it to span across. Remove the cells in the affected rows that correspond to the spanned cell. For example, if you have a cell that spans two rows, you should remove one cell in each of the two rows that would be covered by the spanned cell.

To merge two or more columns into a single column:

Identify the cell that you want to span across multiple columns.
Add the colspan attribute to that cell, and set its value to the number of columns you want it to span across.
Remove the cells in the affected columns that correspond to the spanned cell. For example, if you have a cell that spans two columns, you should remove one cell in each of the two columns that would be covered by the spanned cell.
Note that when you use rowspan or colspan, you need to adjust the number of cells in the affected rows or columns accordingly, so that the table structure remains valid.

Q.16 : How are active links different from normal links ?

Ans. :Active links and normal links are not distinct types of links, but rather they describe the behavior of links when a user interacts with them. A normal link, also known as an inactive link or static link, is a hyperlink that displays a web page or resource when clicked on, but does not change in appearance when a user hovers over it or clicks on it. An active link, also known as a dynamic link or interactive link, responds to user input in some way. For example, an active link may change color or style when a user hovers over it, or it may display a preview of the destination page when a user clicks on it. Active links are generally considered more user-friendly and engaging than normal links because they provide visual feedback and interactivity to the user. They can help guide users through a website or application by highlighting important areas or actions, and can help to make the overall user experience more intuitive and enjoyable.

Q.18 : What is SVG ?

Ans. :Scalable Vector Graphics (SVG) is an XML-based vector image format for defining two-dimensional graphics, having support for interactivity and animation. The SVG specification is an open standard developed by the World Wide Web Consortium since 1999. SVG images are defined in a vector graphics format and stored in XML text files. SVG images can thus be scaled in size without loss of quality, and SVG files can be searched, indexed, scripted, and compressed. The XML text files can be created and edited with text editors or vector graphics editors, and are rendered by the most-used web browsers.

Q.20 : What are logical and physical tags in HTML ?

Ans. :In HTML, tags are used to define the structure and content of a web page. There are two types of tags in HTML: logical tags and physical tags. Logical tags describe the meaning or purpose of the content they surround, while physical tags define how the content should be displayed on the page. Logical tags include header tags such as <h1>, <h2>, <h3>, etc. which are used to define headings and subheadings on a page. Other logical tags include <p> for paragraphs, <ul> and <ol> for lists, <em> and <strong> for emphasizing or highlighting text, and <a> for hyperlinks. Physical tags, on the other hand, describe how the content should be displayed on the page. Examples of physical tags include <div> and <span>, which are used to group and style content, <img> for images, <table> for tables, and <form> for input forms.
